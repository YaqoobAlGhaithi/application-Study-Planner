# تصميم تطبيق التخطيط الدراسي - Study Planner

## نظرة عامة
تطبيق جوال متخصص في التخطيط الدراسي يساعد الطلاب على تحديد أهدافهم التعليمية وتقسيمها إلى مهام قابلة للتنفيذ مع تتبع تقدمهم بشكل منظم وجذاب.

## معايير التصميم
- **الاتجاه**: عمودي (Portrait 9:16)
- **الاستخدام**: بيد واحدة
- **المعايير**: Apple Human Interface Guidelines (HIG)
- **الأسلوب**: تطبيق iOS أول الطرف (First-party iOS app)

---

## قائمة الشاشات

### 1. شاشة الترحيب (Onboarding)
**الهدف**: استقبال المستخدم الجديد وشرح ميزات التطبيق

**المحتوى الرئيسي**:
- شعار التطبيق
- عنوان ترحيبي
- وصف موجز للميزات (3-4 نقاط)
- زر "ابدأ الآن" (CTA رئيسي)

**الوظائف**:
- الانتقال إلى الشاشة الرئيسية

---

### 2. الشاشة الرئيسية (Home Screen)
**الهدف**: عرض ملخص الأهداف والمهام الحالية

**المحتوى الرئيسي**:
- رسالة ترحيب شخصية ("مرحباً، [الاسم]")
- إحصائيات سريعة:
  - عدد الأهداف النشطة
  - عدد المهام المكتملة اليوم
  - نسبة التقدم الإجمالية
- قائمة الأهداف الحالية (كارتات):
  - اسم الهدف
  - نسبة التقدم (Progress bar)
  - عدد المهام المكتملة / الإجمالية
  - تاريخ الانتهاء المتوقع
- زر عائم (FAB) لإضافة هدف جديد

**الوظائف**:
- الضغط على كارت الهدف → الانتقال إلى تفاصيل الهدف
- الضغط على FAB → فتح نموذج إضافة هدف جديد
- السحب لأسفل → تحديث البيانات

---

### 3. شاشة تفاصيل الهدف (Goal Details)
**الهدف**: عرض تفاصيل الهدف الواحد والمهام المرتبطة به

**المحتوى الرئيسي**:
- رأس الشاشة:
  - اسم الهدف
  - نسبة التقدم الكاملة (دائرية أو خطية)
  - تاريخ الانتهاء
- معلومات الهدف:
  - الوصف
  - الفئة (مثل: لغات، برمجة، رياضيات)
  - الأولوية (عالية، متوسطة، منخفضة)
- قائمة المهام الفرعية:
  - كل مهمة كصف مع checkbox
  - اسم المهمة
  - تاريخ الاستحقاق
  - حالة الإنجاز (مكتملة / قيد التنفيذ)
- زر عائم لإضافة مهمة جديدة
- زر تعديل الهدف
- زر حذف الهدف

**الوظائف**:
- الضغط على checkbox → تحديث حالة المهمة
- الضغط على المهمة → فتح تفاصيل المهمة للتعديل
- الضغط على FAB → فتح نموذج إضافة مهمة جديدة
- الضغط على زر التعديل → فتح نموذج تعديل الهدف

---

### 4. شاشة إضافة/تعديل الهدف (Add/Edit Goal)
**الهدف**: السماح للمستخدم بإنشاء أو تعديل هدف تعليمي

**المحتوى الرئيسي**:
- نموذج بالحقول التالية:
  - اسم الهدف (نص)
  - الوصف (نص طويل)
  - الفئة (قائمة منسدلة)
  - الأولوية (أزرار راديو)
  - تاريخ الانتهاء المتوقع (منتقي التاريخ)
- زر "حفظ"
- زر "إلغاء"

**الوظائف**:
- التحقق من صحة البيانات
- حفظ الهدف في التخزين المحلي
- الرجوع إلى الشاشة السابقة

---

### 5. شاشة إضافة/تعديل المهمة (Add/Edit Task)
**الهدف**: السماح بإضافة أو تعديل مهمة فرعية

**المحتوى الرئيسي**:
- نموذج بالحقول التالية:
  - اسم المهمة (نص)
  - الوصف (نص طويل - اختياري)
  - تاريخ الاستحقاق (منتقي التاريخ)
  - الأولوية (أزرار راديو)
- زر "حفظ"
- زر "إلغاء"

**الوظائف**:
- التحقق من صحة البيانات
- حفظ المهمة
- الرجوع إلى تفاصيل الهدف

---

### 6. شاشة الإحصائيات (Statistics)
**الهدف**: عرض تقارير مفصلة عن التقدم الدراسي

**المحتوى الرئيسي**:
- رسوم بيانية:
  - نسبة إنجاز الأهداف (دائري)
  - توزيع المهام حسب الحالة (أعمدة)
  - التقدم الأسبوعي (خطي)
- جداول ملخصة:
  - عدد الأهداف: نشطة / مكتملة / متأخرة
  - عدد المهام: مكتملة / قيد التنفيذ / متأخرة
- خيارات التصفية:
  - حسب الفترة الزمنية (أسبوع / شهر / سنة)
  - حسب الفئة

**الوظائف**:
- تحديث الرسوم البيانية عند تغيير البيانات
- تصدير التقارير (مستقبلاً)

---

### 7. شاشة الإعدادات (Settings)
**الهدف**: إدارة تفضيلات التطبيق

**المحتوى الرئيسي**:
- قسم الحساب:
  - اسم المستخدم
  - البريد الإلكتروني
- قسم الإشعارات:
  - تفعيل/تعطيل الإشعارات
  - وقت الإشعارات الافتراضي
- قسم المظهر:
  - اختيار المظهر (فاتح / داكن / تلقائي)
- قسم البيانات:
  - حذف جميع البيانات
  - تصدير البيانات
- قسم المساعدة:
  - حول التطبيق
  - الإصدار

**الوظائف**:
- حفظ التفضيلات في التخزين المحلي
- تطبيق المظهر فوراً

---

## تدفقات المستخدم الرئيسية

### 1. تدفق إنشاء هدف جديد
```
الشاشة الرئيسية 
  → الضغط على FAB 
  → شاشة إضافة الهدف 
  → ملء البيانات 
  → الضغط على "حفظ" 
  → العودة إلى الشاشة الرئيسية (مع ظهور الهدف الجديد)
```

### 2. تدفق إضافة مهام إلى هدف
```
الشاشة الرئيسية 
  → الضغط على كارت الهدف 
  → شاشة تفاصيل الهدف 
  → الضغط على FAB 
  → شاشة إضافة المهمة 
  → ملء البيانات 
  → الضغط على "حفظ" 
  → العودة إلى تفاصيل الهدف (مع ظهور المهمة الجديدة)
```

### 3. تدفق تحديث حالة المهمة
```
شاشة تفاصيل الهدف 
  → الضغط على checkbox للمهمة 
  → تحديث الحالة فوراً 
  → تحديث نسبة التقدم
```

### 4. تدفق عرض الإحصائيات
```
التنقل من خلال Tab Bar 
  → الضغط على تبويب "الإحصائيات" 
  → عرض الرسوم البيانية والملخصات
```

## مبادئ التصميم

### 1. البساطة والوضوح
- واجهة نظيفة وخالية من الفوضى
- رسائل واضحة ومباشرة
- تسلسل هرمي منطقي للمعلومات

### 2. سهولة الاستخدام
- تفاعلات بديهية
- تعليقات فورية على الإجراءات
- دعم الملاحة الخلفية

### 3. الاستجابة البصرية
- تأثيرات ضغط خفيفة على الأزرار
- حركات انتقالية سلسة
- تحديثات حالة فورية

### 4. إمكانية الوصول
- حجم نص مناسب (14-16px للنص الأساسي)
- تباين ألوان كافٍ
- دعم الوضع الداكن

## ملاحظات تقنية

- **التخزين**: AsyncStorage للبيانات المحلية
- **الحالة**: React Context + useReducer
- **الرسوم البيانية**: مكتبة Plotly أو Chart.js
- **التنقل**: Expo Router مع Tab Bar
- **الأيقونات**: Material Icons من Expo Vector Icons
